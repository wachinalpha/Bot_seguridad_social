# 0. Registrar decisiones de arquitectura

Fecha: 2026-02-05

## Estado

Aceptado

## Contexto

Necesitamos registrar las decisiones arquitectónicas importantes para este proyecto, de modo que no perdamos el contexto de por qué se tomaron ciertas decisiones cuando se unan nuevos miembros (el junior, agencias nuevas) o cuando revisemos el código meses después.

## Decisión

Usaremos Architecture Decision Records (ADRs) para registrar estas decisiones.

Esta es una técnica simple donde cada decisión se registra en un archivo de texto separado en una carpeta `docs/adr`.

## Consecuencias

Tendremos un historial claro de decisiones. Requiere disciplina escribir estos registros, pero el costo de escritura es bajo comparado con el costo de perder contexto.
